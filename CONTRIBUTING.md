## Reporting Bugs

If you've found a bug, please help us fix it by providing detailed information:

### Before Submitting a Bug Report

1. **Check existing issues**: Search the issue tracker to see if the bug has already been reported
2. **Verify the bug**: Make sure you can reproduce the issue consistently
3. **Test with the latest version**: Update to the latest version to see if the bug has been fixed

### What to Include in a Bug Report

Please provide as much of the following information as possible:

- **Description**: Clear and concise description of the bug
- **Steps to reproduce**: Exact steps to trigger the issue
- **Expected behavior**: What you expected to happen
- **Actual behavior**: What actually happened
- **Version information**:
  - Extension version (check `CHANGELOG.md` or `pyproject.toml`)
  - ComfyUI version
  - Python version
  - Operating system and version
- **Command used**: The exact command that caused the issue (redact sensitive data like API tokens)
- **Error output**: Complete error logs from ComfyUI
- **Additional context**: Any other relevant information (screenshots, logs, etc.)

## Suggesting Features

We welcome feature requests! Before suggesting a new feature, check in the bug tracker if the feature has already been requested.

### What to Include in a Feature Request

- **Use case**: Describe the problem you're trying to solve
- **Proposed solution**: How you envision the feature working
- **Alternative solutions**: Other approaches you've considered
- **Example usage**: How the feature would look and how it could integrate in a typical workflow

## Code Guidelines

- Follow [PEP 8](https://pep8.org/) style guidelines for Python code
- Follow [JavaScript Standard Style](https://standardjs.com/) guidelines for Javascript code
- Use meaningful variable and function names
- Keep functions focused and modular
- Add docstrings to functions and classes
- Use type hints where appropriate

## Submitting Changes

External contributions are accepted and appreciated! Create a Pull Request to merge your code on the main branch.

### Pull Request Guidelines

- **One feature per PR**: Keep changes focused and reviewable
- **Clear title and description**: Explain the motivation and implementation, what changed and why, reference related issues
- **Updated docs**: Include documentation updates in the same PR
- **Clean history**: Squash or clean up commits if needed
- **Test coverage**: Describe how you tested the changes
